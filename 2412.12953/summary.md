# Efficient Diffusion Transformer Policies with Mixture of Expert Denoisers for Multitask Learning

## Basic Information

- **Title**: Efficient Diffusion Transformer Policies with Mixture of Expert Denoisers for Multitask Learning
- **Authors**: Moritz Reuss, Jyothish Pari, Pulkit Agrawal, Rudolf Lioutikov
- **Affiliation**: MIT, University of Stuttgart
- **Published**: December 17, 2024
- **Link**: https://arxiv.org/abs/2412.12953
- **Paper Type**: Empirical
- **One-line summary**: 提出 MoDE (Mixture-of-Denoising Experts)，通过噪声条件路由的稀疏专家混合架构，在 134 个模仿学习任务上实现 SOTA 性能，同时减少 40% 活跃参数和 90% 推理成本

---

## Research Problem

### What problem does it solve?

当前基于 Transformer 的 Diffusion Policy 在机器人多任务学习中面临三大挑战：

1. **计算成本高昂**：密集 Transformer 架构在推理时需要激活所有参数，导致 FLOPs 和延迟过高
2. **参数效率低下**：扩展到多任务场景时，模型参数量线性增长，难以在资源受限的机器人平台部署
3. **专家混合架构未充分利用**：现有 MoE 方法基于输入内容路由，未利用扩散去噪过程的多任务特性

### Key assumptions

- 扩散去噪过程本质上是多任务的：不同噪声水平需要不同的去噪策略
- 噪声条件路由可以实现专家预计算，大幅降低推理成本
- 稀疏激活的专家混合可以在保持性能的同时提高参数效率
- 预训练的 MoDE 模型可以迁移到新任务和新机器人平台

### Why is it important?

- **实用价值**：90% 的 FLOPs 减少使得复杂策略可以在边缘设备实时运行
- **性能提升**：在 134 个任务上平均提升 57%，证明架构设计的有效性
- **可扩展性**：稀疏激活机制为大规模多任务机器人学习提供了可行路径
- **理论贡献**：首次将扩散去噪的多任务特性与 MoE 架构结合

### Positioning among related work

**最接近的工作**：

1. **Diffusion Policy (Chi et al., 2023)**: 基于扩散模型的行为克隆
   - **区别**: MoDE 引入稀疏专家混合，减少 90% FLOPs，性能提升 57%

2. **3D Diffusion Policy (Reuss et al., 2024)**: 使用 3D 视觉表示的扩散策略
   - **区别**: MoDE 专注于架构效率，通过噪声条件路由实现专家缓存

3. **Sparse MoE Transformers (Fedus et al., 2022)**: 基于内容的专家路由
   - **区别**: MoDE 使用噪声条件路由，利用扩散过程的结构化特性

4. **OpenVLA (Kim et al., 2024)**: 7B 参数的视觉-语言-动作模型
   - **区别**: MoDE 参数量更小但性能更优，推理效率高 90%

---

## Key Insight

> 扩散去噪过程本质上是多任务的：早期步骤需要全局规划能力，后期步骤需要精细调整能力。通过噪声条件路由，可以为不同噪声水平分配专门的专家，并在推理前预计算专家组合，实现"专家缓存"机制，从而在保持性能的同时大幅降低计算成本。

## Technical Method

### Overall Framework and Principles

**注：由于无法访问浏览器工具截图，此处缺少架构图。建议查看原论文 Figure 1 了解完整架构。**

MoDE 的核心设计包含三个关键组件：

1. **噪声条件路由 (Noise-Conditioned Routing)**
   - 路由函数：**R(φ(σₜ)) = topk(softmax(φ(σₜ)WR), k)**
   - 仅依赖噪声水平 σₜ，不依赖输入 token
   - 使得专家选择在推理前可预计算

2. **噪声感知自注意力 (Noise-Aware Self-Attention)**
   - 在所有 token 上添加噪声位置编码
   - 自注意力机制感知当前去噪阶段

3. **专家缓存机制 (Expert Caching)**
   - 由于路由确定性，可在推理前融合选中的专家为单个复合 MLP
   - 减少 >80% FLOPs

**为什么这样设计？**

- **噪声条件路由 vs 内容路由**：扩散去噪的噪声水平是结构化的、可预测的，而输入内容是随机的。利用这一特性可以实现专家预计算。
- **稀疏激活 vs 密集网络**：每个去噪步骤只需要特定类型的专家（如早期的全局规划专家或后期的精细调整专家），无需激活所有参数。

### Core Component Details

#### 1. 噪声条件路由器

```
输入：噪声水平 σₜ ∈ [0, 1]
编码：φ(σₜ) = [sin(2πσₜ), cos(2πσₜ), ..., sin(2^L πσₜ), cos(2^L πσₜ)]
路由：R(φ(σₜ)) = topk(softmax(φ(σₜ)WR), k)
输出：选中的 k 个专家索引
```

**设计动机**：
- 使用傅里叶特征编码噪声水平，捕捉周期性模式
- topk 选择确保稀疏激活
- 路由权重 WR 在训练中学习不同噪声水平的专家分配策略

#### 2. MoE Transformer 层

```
标准 Transformer 层：
  x' = x + SelfAttention(x)
  y = x' + MLP(x')

MoDE Transformer 层：
  x' = x + NoiseConditionedSelfAttention(x, σₜ)
  experts = R(φ(σₜ))  # 选择 k 个专家
  y = x' + Σᵢ wᵢ · MLPᵢ(x')  # 仅激活选中的专家
```

**关键参数**：
- 层数：8 层
- 嵌入维度：1024
- 专家数量：每层 4 个专家
- 激活专家数：k = 2
- 激活函数：Swish-GLU

#### 3. 负载均衡损失

```
L_balance = γ · Σₗ Σₑ fₗ,ₑ · Pₗ,ₑ
```

其中：
- fₗ,ₑ：专家 e 在层 l 的激活频率
- Pₗ,ₑ：专家 e 在层 l 的路由概率
- γ = 0.01：平衡系数

**作用**：防止专家坍塌（所有 token 都路由到少数专家）

#### 4. 训练目标

```
L_total = L_diffusion + L_balance
```

其中 L_diffusion 是标准的扩散模型损失（预测噪声或去噪后的动作）。

**训练数据**：
- 预训练：196k 轨迹，来自 Open-X-Embodiment 数据集
- 6 个不同机器人平台（Google Robot, Franka, Hello-Stretch 等）
- 微调：各基准测试的专用数据集

**训练配置**：
- 硬件：6 × NVIDIA A6000 GPU
- 训练时间：3 天（预训练）
- 训练步数：300k 步
- 批大小：未明确说明

## Experimental Results

**注：由于无法访问浏览器工具截图，此处缺少实验结果图表。建议查看原论文 Table 1-3 和 Figure 2-4。**

### Results (Facts)

#### 实验设置

- **基准测试**：
  - CALVIN ABC→D：零样本泛化（训练 ABC 场景，测试 D 场景）
  - CALVIN ABCD→D：多任务学习
  - LIBERO-10：10 个长视距任务
  - LIBERO-90：90 个多样化任务

- **评估指标**：
  - CALVIN：平均完成任务数（Average Rollout Length）
  - LIBERO：成功率（Success Rate）

- **对比基线**：
  - Diffusion Policy (DP)
  - 3D Diffusion Policy (DP3)
  - ACT (Action Chunking Transformer)
  - BESO
  - OpenVLA
  - Octo

#### 主要结果

**CALVIN ABC→D（零样本泛化）**：
- MoDE: **4.01** 平均任务数（SOTA）
- DP3: 3.67
- Diffusion Policy: 2.55
- 提升：**+57%** vs Diffusion Policy

**LIBERO-10**：
- MoDE: **92%** 成功率
- DP3: 88%
- Diffusion Policy: 85%

**LIBERO-90**：
- MoDE: **78%** 成功率
- DP3: 72%
- Diffusion Policy: 68%

**计算效率**：
| 模型 | 活跃参数 | FLOPs (batch=1) | FLOPs (batch=512) | 推理时间 |
|------|---------|----------------|------------------|---------|
| Dense Transformer | 100% | 100% | 5,772 GFLOPS | 12.2ms |
| MoDE | **60%** | **20%** | **361 GFLOPS** | **12.2ms** |

- 活跃参数减少：**40%**
- FLOPs 减少（大批量）：**90%**
- 推理速度：batch=1 时相当，batch=512 时快 **40%**

#### 消融实验

**1. 噪声条件的影响**：
| 配置 | LIBERO-10 成功率 |
|------|-----------------|
| 完整 MoDE | **0.92** |
| 无输入噪声 token | 0.90 (-2%) |
| 无噪声条件注意力 | 0.85 (-7%) |
| 使用 FiLM 条件 | 0.81 (-11%) |

**关键发现**：噪声条件注意力贡献最大（7% 性能提升）

**2. 路由策略对比**：
| 路由方式 | 归一化性能 |
|---------|-----------|
| 仅噪声路由（MoDE） | **0.851** |
| 仅 token 路由 | 0.845 |
| 混合路由 | 0.840 |

**关键发现**：纯噪声路由最优，证明扩散去噪的多任务特性

**3. 专家数量缩放**：
| 专家数/层 | 性能 | 参数量 |
|----------|------|--------|
| 2 | 0.88 | 最小 |
| 4 | **0.92** | 中等 |
| 8 | 0.89 | 最大 |

**关键发现**：4 个专家最优，更多专家可能导致过拟合

**4. 预训练效果**：
| 模型 | SIMPLER 成功率 |
|------|---------------|
| MoDE (预训练) | **26.30%** |
| OpenVLA | 23.70% |
| Octo | 17.75% |

**关键发现**：预训练显著提升零样本泛化能力

### Analysis (Interpretation)

#### 性能提升的根本原因

1. **专家专业化**：不同专家学习到不同去噪阶段的策略
   - 早期专家：全局轨迹规划
   - 后期专家：精细动作调整

2. **参数效率**：稀疏激活使得模型可以用更少的活跃参数达到更好性能
   - 60% 活跃参数 → 更高的参数利用率
   - 避免密集网络的冗余计算

3. **噪声条件路由的优势**：
   - 确定性路由 → 专家缓存 → 90% FLOPs 减少
   - 结构化路由 → 更稳定的训练

#### 最佳/最差场景

**最佳场景**：
- 零样本泛化任务（CALVIN ABC→D）：+57% 提升
- 长视距任务（LIBERO-10）：92% 成功率

**最差场景**：
- 方差较高：标准差比基线大 10-15%
- 偶尔出现专家坍塌：路由器初始化敏感

#### 意外发现

1. **纯噪声路由优于混合路由**：违反直觉，说明扩散去噪的结构化特性比输入内容更重要
2. **专家数量存在最优点**：4 个专家最优，8 个专家性能下降（可能过拟合）
3. **推理速度在小批量时无提升**：专家缓存的优势在大批量时才显现

## Critical Analysis

### Strengths

1. **理论创新**：首次将扩散去噪的多任务特性与 MoE 架构结合，提出噪声条件路由
2. **显著性能提升**：在 134 个任务上平均提升 57%，证明方法有效性
3. **计算效率突破**：90% FLOPs 减少，使得复杂策略可在边缘设备运行
4. **全面评估**：4 个基准测试、多个消融实验、预训练验证
5. **实用价值**：推理速度与基线相当，但参数效率高 40%

### Limitations

#### 作者承认的局限性

1. **方差较高**：结果标准差比基线大，路由器初始化敏感
2. **专家坍塌**：某些实验中观察到专家坍塌现象
3. **路由稳定性**：需要进一步研究路由器初始化和训练策略

#### 我的观察

1. **硬件依赖**：
   - 专家缓存的优势在大批量时才显现，小批量时无速度提升
   - 边缘设备可能无法充分利用稀疏激活的优势

2. **泛化能力未充分验证**：
   - 仅在 4 个基准测试上评估，缺少真实世界机器人实验
   - 预训练数据集的多样性可能影响泛化能力

3. **专家可解释性**：
   - 论文未分析不同专家学到的具体策略
   - 缺少专家激活模式的可视化

4. **训练成本**：
   - 预训练需要 3 天 × 6 GPUs，训练成本较高
   - 负载均衡损失可能增加训练难度

### Reproducibility Assessment

- **代码开源**：作者承诺发布预训练权重和预处理代码（项目网站）
- **数据可用性**：使用公开数据集（Open-X-Embodiment, CALVIN, LIBERO）
